## Usage in Components

### Basic Translation

```vue
<script setup lang="ts">
const { t } = useI18n()
</script>

<template>
  <h1>{{ t('pages.contractors.title') }}</h1>
  <Button :label="t('buttons.add')" />
</template>
```

### Translation with Parameters

```typescript
// In locale file
{
  "validation": {
    "minLength": "Minimum {count} znaków",
    "between": "Wartość musi być między {min} a {max}"
  }
}

// In component
const errorMessage = t('validation.minLength', { count: 5 })
// Output: "Minimum 5 znaków"

const rangeMessage = t('validation.between', { min: 1, max: 100 })
// Output: "Wartość musi być między 1 a 100"
```

### Pluralization

```typescript
// In locale file
{
  "items": {
    "selected": "{count} wybrany | {count} wybrane | {count} wybranych"
  }
}

// In component
t('items.selected', 1)  // "1 wybrany"
t('items.selected', 2)  // "2 wybrane"
t('items.selected', 5)  // "5 wybranych"
```

## Common Patterns

### Page Titles

```vue
<script setup lang="ts">
const { t } = useI18n()

useHead({
  title: t('pages.contractors.title'),
})
</script>
```

### Form Labels

```vue
<template>
  <div class="field">
    <label for="name">
      {{ t('fields.name') }}
      <span class="text-red-500">*</span>
    </label>
    <InputText
      id="name"
      v-model="name"
      :placeholder="t('pages.contractors.searchPlaceholder')"
    />
  </div>
</template>
```

### Toast Messages

```typescript
const { t } = useI18n()
const toast = useToaster()

// Success
toast.success(t('toast.success.created'))

// Error
toast.error(t('toast.error.generic'))

// With parameters
toast.success(t('contractor.deleteSuccess', { name: contractor.name }))
```

### Confirmation Dialogs

```typescript
confirm.require({
  message: t('contractor.confirmDelete.message', { name: contractor.name }),
  header: t('contractor.confirmDelete.header'),
  acceptLabel: t('buttons.delete'),
  rejectLabel: t('buttons.cancel'),
})
```

## Best Practices

### DO

1. ✅ Always use `t('key')` for user-facing text
2. ✅ Group translations by domain/page
3. ✅ Use descriptive key names
4. ✅ Use parameters for dynamic values
5. ✅ Keep translations in `src/static/locales/`
6. ✅ Use consistent naming conventions
7. ✅ Extract common strings to `common` section
8. ✅ Use nested objects for organization

### DON'T

1. ❌ Hardcode user-facing text
2. ❌ Use English in Polish locale
3. ❌ Concatenate translated strings
4. ❌ Use long keys (keep them readable)
5. ❌ Duplicate translations
6. ❌ Mix languages in one file
7. ❌ Forget to translate error messages
8. ❌ Use translation keys as fallback text

## Key Naming Conventions

### Pattern: `[domain].[subdomain].[key]`

```
pages.contractors.title
pages.contractors.add
fields.name
fields.email
buttons.save
buttons.cancel
toast.success.created
toast.error.generic
validation.required
common.loading
```

## Testing with i18n

```typescript
import { mount } from '@vue/test-utils'
import { describe, it, expect } from 'vitest'

describe('ContractorForm', () => {
  const wrapper = mount(ContractorForm, {
    global: {
      mocks: {
        $t: (key: string) => key, // Mock translation
      },
    },
  })

  it('should display translated title', () => {
    expect(wrapper.text()).toContain('pages.contractors.title')
  })
})
```
